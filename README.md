
 需求拆解:
 
     信息：
     1. 类别划分：区域、初始化位置
        1.1 区域：长X+宽Y
        1.2 初始化：降落坐标（x+y）、朝向（E|S|W|N）
    
     指令：
     1. 指令类别划分：移动、转向
        1.1 移动：前进f|后退b
        1.1 转向：左转90°|右转90°
     2. 指令发送方式：批量发送
    
     火星车：
     1. 执行批量命令
     2. 回报自己所在位置坐标（x+y）、朝向（E|S|W|N）



 针对初始需求，需要注意点：
 
     1. 区域限制：
        1.1 若火星车初始化坐标在区域边界或区域外，则不执行移动指令
        1.2 若火星车在移动过程中行驶到区域边界，则不再继续执行移动指令
    
     2. 朝向：认为初始化朝向只能是四个方向，不能有复合方向，比如东偏南等
